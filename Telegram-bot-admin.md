# 🤖 **Telegram-бот администратор** [Salebot]

## **Для чего создавался**
*компания в b2b сегменте, поэтому коммуницирует с большим числом юридических лиц (более 10.000). 
Для каждого юр. лица заводится отдельная телеграм-группа для удобства работы и в неё же добавляется телеграм-бот.
Однако, основные рабочие процессы с компанией (хранение данных, документооборот и т.д.) протекает в рамках CRM.
Из этого вытекают следующие потребности у бизнеса, который закрывает данный телеграм-бот:*

1. Массовые информационные рассылки по группам
2. Маппинг между сервисами (сопоставление данных о компании в CRM и группами в телеграме). 
В данном случае, между собой синхронизируются CRM Bitrix24 <-> Salebot по REST API. Маппинг позволяет сегментировать
группы, отталкиваясь от данных в CRM, например, по определённым полям в компаниях. Это позволяет более гибко настраивать
автоматизацию по каждому сегменту.
3. Автоматизация и обновление данных в CRM (процессы, стадии, поля и т.д.)
4. Автоматизация в группах телеграм (смена названия группы, аватара, редактирование и удаление сообщений, блокировки пользователей и т.д.)

Последовательность автоматизации:

1: Определяем в каком мессенджере группа;
2: API запросы на получение компании из CRM Bitrix24, поиск осуществляется по полю с user_id из телеграм (id группы);
3. Сегментация по спискам в salebot в зависимости от того получили id компании или нет;
4. В полях компании в CRM обновляем информацию, добавляем ссылку для другого сервиса (Chatapp);
5. Обработка ошибки, добавление в список с ошибкой формирования ссылки;
6. Обновляем данные в CRM, добавляем пригласительную ссылку для вступления в группу в CRM;
7. Дополнительный API запрос в CRM c получением данных по определенным полям. После получения ответа ветвим логику в зависимости от условий
и далее распределяем группы по спискам в Salebot.

## **Как итог имеем следующие результаты автоматизации:**
- Сотрудники не заходят по очереди в каждую группу и не копируют пригласительную ссылку чтобы потом добавить в CRM, система делает это автоматически;
- Менеджеры отдела продаж не делают ручные рассылки по своим партнёрам, группы отсегментированы по ответственным менеджерам в отделе продаж и рассылка по определенному сегменту
делается в пару кликов;
- Сотрудникам не нужно заводить несколько аккаунтов чтобы преодолеть лимиты нахождения в группах - бот уже находится в более чем 10.000 группах;

## **Описание дополнительной воронки автоматизации:**

Схема в конструкторе:
<img width="1399" height="954" alt="Скриншот 22-06-2026 210547" src="https://github.com/user-attachments/assets/44841298-10e4-4013-8286-8b107b8f09e6" />

Последовательность автоматизации:

1: Отслеживаем текстовый триггер "#звонок" в чате группы
2: На этом этапе нам необходимо получить id лида для поиска в CRM. Его могут прислать двумя вариантами:
- Первый: пользователь отвечает (текстом #звонок) на сообщение в котором уже есть id лида. В этом случае просто режем строку, т.к. точно знаем где он находится.
- Второй: пользователь пишет триггер #звонок и прописывает id лида, в этом случае через python скрипт дёргаем id:
```
import json

def handle(data):
    data = json.loads(data)
    text = data['result']

    num = ""
    for ch in text:
        if "0" <= ch <= "9":
            num += ch
        elif num:
            break

    return {"lead_id": num}
```
3: Через API запрос ищем сделку в CRM. 
4. Если сделка найдена, то в зависимости от гражданства клиента запускаем автоматизацию на конкретной стадии - сотрудник уведомляется о том, что необходимо
связаться с клиентом.
5. Если сделка не найдена, то ищем лид в CRM и выполняем автоматизацию аналогично п.4
6. Обработка исключения - если сделка или лид не найдены, то пользователь явно допустил ошибку и система просит ввести id повторно
7. Уведомляем сотрудника в группе bitrix24.

## **Как итог имеем следующие результаты автоматизации:**
- Пользователь в группе сам при необходимости запускает автоматизацию в CRM, сотрудникам не приходится мониторить группы или открытые линии в битриксе, система уведомляет автоматически
когда необходимо связаться с клиентом.
- Система сама меняет стадии лида или сделки, если это требуется.
- Сотрудники уведомляются в общем чате в битриксе, при этом система заранее проверяет кто был ответственным в лиде или сделке и уведомляет именно этого сотрудника.
